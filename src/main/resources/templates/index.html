      • Testes Automatizados

        - O que são testes?
                  Teste de software geralmente é a última etapa na construção de um programa, visando checar o seu nível de qualidade. Os defeitos que um teste busca identificar incluem erro de compatibilidade, de algum algoritmo, de requisitos que não podem ser complementados, limitação de hardware etc. A lista é grande e aumenta com o tamanho do programa.
        Ver mais

        - O que são testes automatizados?
                  Testes automatizados se fundamentam no uso de ferramentas para controlar a execução de testes de software. Através da aplicação de estratégias, o objetivo é basicamente escrever um software que testará seu programa por você.A automação permite que o teste seja repetido várias vezes, sendo mais fácil encontrar novos erros através da repetição e da simulação de cenários específicos. O objetivo final dos testes automatizados é minimizar os problemas da abordagem manual, o tempo despendido e, consequentemente, o custo final.
        Ver mais

        - Por que automatizar testes?
                  A automação de testes está diretamente atrelada ao que é esperado do produto final. Esse é um passo que deve ser dado visando principalmente a viabilidade do processo. É importante que com a automação de testes de software você ganhe tempo, mantenha a qualidade e reduza os custos. Além disso, tenha em mente a quantidade de vezes que esses testes devem ser realizados e com qual frequência serão feitos novamente, o que é esperado quanto aos testes obtidos e o grau de mudança das funcionalidades que irão para teste. Afinal, não é interessante automatizar o teste de uma funcionalidade que vai ser alterada num intervalo pequeno de tempo.
        Ver mais

        - Caso de testes:
                  Um caso de teste responde a pergunta: "O que eu irei testar?" Você desenvolve casos de teste para definir o que deve ser validado para garantir que o sistema esteja funcionando corretamente e esteja compilado com um alto nível de qualidade.
        Ver mais

        - Suite de testes:
                  Um Suíte de Testes é uma coleção de casos de testes que são agrupados para propósitos de execução de teste. Os suítes de teste podem identificar as diferenças em um esforço de teste, quando a conclusão bem-sucedida de um caso de teste deve ocorrer antes que você inicie o próximo caso de teste. Por exemplo, não é possível incluir novos produtos em um carrinho antes de efetuar login com sucesso no aplicativo. Ao executar um suíte de testes no modo sequencial, é possível optar por parar a execução do suíte de testes se um único caso de teste não passar. Parar a execução será útil se a execução de um caso de teste em um Suíte de Testes depender do êxito dos casos de teste anteriore

      • API

        - O que é API?
                  API é um conjunto de rotinas e padrões de programação para acesso a um aplicativo de software ou plataforma baseado na Web. A sigla API refere-se ao termo em inglês "Application Programming Interface" que significa em tradução para o português "Interface de Programação de Aplicativos". Uma API é criada quando uma empresa de software tem a intenção de que outros criadores de software desenvolvam produtos associados ao seu serviço. Existem vários deles que disponibilizam seus códigos e instruções para serem usados em outros sites da maneira mais conveniente para seus usuários. O Google Maps é um dos grandes exemplos na área de APIs. Por meio de seu código original, muitos outros sites e aplicações utilizam os dados do Google Maps adaptando-o da melhor forma a fim de utilizar esse serviço.
        Ver mais

        - Protocolo HTTP:
                  HTTP é um protocolo (protocol) que permite a obtenção de recursos, como documentos HTML. É a base de qualquer troca de dados na Web e um protocolo cliente-servidor, o que significa que as requisições são iniciadas pelo destinatário, geralmente um navegador da Web. Um documento completo é reconstruído a partir dos diferentes sub-documentos obtidos, como por exemplo texto, descrição do layout, imagens, vídeos, scripts e muito mais. Clientes e servidores se comunicam trocando mensagens individuais (ao contrário de um fluxo de dados). As mensagens enviadas pelo cliente, geralmente um navegador da Web, são chamadas de solicitações (requests), ou também requisições, e as mensagens enviadas pelo servidor como resposta são chamadas de respostas (responses).
        Ver mais

        - Frontend:
                  O desenvolvimento de front end é a parte do desenvolvimento da web que codifica e cria elementos de front end de um site, que são recursos diretamente visíveis e acessíveis pelo usuário final ou cliente. Essencialmente, um desenvolvedor front-end é responsável por tudo o que você vê e trabalha para aprimorar a experiência do usuário para garantir que seja perfeita. Eles ajudam a criar o design geral e a estética, além de depurar e usar a análise estática de código.
        Ver mais

        - Backend:
                  O desenvolvimento de back-end é apropriadamente nomeado para o desenvolvimento da web que ocorre no back-end dos programas. Em contraste com o desenvolvimento de front-end, que se concentra em produtos e programas voltados para o cliente, o desenvolvimento de back-end aborda a lógica e a integração de aplicativos da Web do lado do servidor. Os desenvolvedores de back-end escrevem código para ajudar um banco de dados e um aplicativo a se comunicarem. Essencialmente, um desenvolvedor de back-end lida com o que você não vê; eles são responsáveis ​​pelo back-end de um site, que inclui servidores, bancos de dados e aplicativos.
        Ver mais

        - Fullstack
                  O termo full-stack refere-se à totalidade de um sistema de computador ou aplicativo. Os desenvolvedores full-stack têm experiência e conhecimento em desenvolvimento web front-end e back-end. Eles são proficientes em todos os estágios de desenvolvimento de software; dos conceitos iniciais ao design UX, implementação, operações e coleta de dados, os desenvolvedores full-stack fazem tudo! Para produzir um produto acabado, os desenvolvedores full-stack usam seu conhecimento de várias linguagens de programação como HTML, CSS, JavaScript, Ruby e muito mais.

      • Postman

        - O que é postman?
                  O Postman é um aplicativo com a função de testar e desenvolver APIs em uma interface bastante simples e intuitiva. Ele nos permite simular requisições HTTP de forma rápida, armazenando-as para que possamos usá-las posteriormente. Além disso, para cada requisição feita, o Postman analisa as respostas enviadas pela API e as exibe visualmente de forma muito agradável e fácil de entender, o que reduz consideravelmente o tempo necessário para o desenvolvimento e testes de sua aplicação

        Ver mais

        - Para que serve?
                  Você pode escrever scripts de teste para suas solicitações de API do Postman em JavaScript. Os testes permitem que você garanta que sua API está funcionando conforme o esperado, para estabelecer se as integrações entre os serviços estão funcionando de forma confiável e para verificar se os novos desenvolvimentos não quebraram nenhuma funcionalidade existente. Você também pode usar o código de teste para ajudar no processo de depuração quando algo der errado com seu projeto de API.
